Overview

High-level flow for the plugin and how it fits inside your OpenClaw deployment.

  1. User sends a message in Pinto.
  2. Pinto calls the OpenClaw webhook at /plugins/pinto/webhook.
  3. The plugin forwards the request to an OpenClaw agent.
  4. The plugin posts the final reply back to Pinto with POST /v1/bots/webhook/receive.
Package pinto-app-openclaw Channel ID pinto

Features

Core behavior included in the current implementation.

Direct Chat Support

Receives Pinto chat events and forwards them into an OpenClaw agent pipeline.

Media Reply Support

Supports outbound replies with optional media_url when the agent returns rich content.

Shared Secret Validation

Handles inbound and outbound X-Pinto-Secret headers to enforce webhook trust.

Requirements

Environment and data you need before installation.

  • OpenClaw instance already running
  • Node.js 20+ and npm
  • Existing Pinto bot
  • Real Pinto bot ID
  • Valid Pinto API base URL
  • Public or reachable webhook URL

OpenClaw Config

Configure via the OpenClaw UI at Channels > Pinto Chat or through the config file.

Field Meaning
Api Url Pinto API base URL. Trailing slash is optional.
Bot Id Must match the real Pinto bot_id value.
Enabled Turns the channel on or off.
Webhook Secret Optional shared secret used in the X-Pinto-Secret header. It is empty by default.
json 
{
  "channels": {
    "pinto": {
      "enabled": true,
      "apiUrl": "https://api-dev.pinto-app.com",
      "botId": "bot-sales",
      "webhookSecret": ""
    }
  }
}

Pinto Config

Fields Pinto needs so it can call OpenClaw correctly.

webhook_url

https://your-host.example.com/plugins/pinto/webhook

Bot ID

Use the Pinto bot bot_id as the OpenClaw Bot Id.

Recommended Flow

Minimal path to a working integration.

  1. Install the plugin.
  2. Restart OpenClaw or reload plugins.
  3. Fill in Api Url, Bot Id, and Webhook Secret.
  4. Set Pinto webhook_url to /plugins/pinto/webhook.
  5. Test the webhook with curl.
  6. Send a real message from Pinto and verify the reply returns.

Webhook Secret

The plugin supports shared-secret validation in both directions.

Inbound

Pinto must include X-Pinto-Secret when OpenClaw config sets a secret. Mismatch returns 401 Unauthorized.

Outbound

Replies back to Pinto automatically include X-Pinto-Secret when configured.

Optional

If no secret is configured, inbound validation is not enforced.

Testing

Smoke-test the integration locally, publicly, and directly against Pinto.

Local Inbound

bash 
curl -i -X POST http://127.0.0.1:18789/plugins/pinto/webhook \
  -H 'Content-Type: application/json' \
  -H 'X-Pinto-Secret: pinto-oc-9f3a1b7c5d2e8k4m' \
  -d '{
    "bot_id":"bot-sales",
    "chat_id":"5f315d4e-cf22-4054-bbb0-2fe074bd3892",
    "message":"hello",

Troubleshooting

Most common integration problems and what they usually mean.

404 Not Found

  • Request used GET instead of POST
  • Reverse proxy or tunnel is not forwarding correctly
  • Path is not /plugins/pinto/webhook

401 Unauthorized

  • X-Pinto-Secret values do not match
  • Pinto and OpenClaw are using different secrets
  • Outbound secret to Pinto is incorrect

No Reply in Pinto

  • Api Url points to the wrong environment
  • Bot Id or chat_id is invalid
  • Pinto API rejected the outbound webhook request
